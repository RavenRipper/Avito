## [BUG-010] Отсутствие ограничений на длину поля `name`


**Окружение**  
- Python 3.8+
- pytest
- Host: `https://qa-internship.avito.com`


**Шаги воспроизведения**  
1. Отправьте POST‑запрос к эндпоинту `/api/1/item` с телом, где поле `name` содержит 10 000 символов:
   ```json
   {
     "sellerID": 111111,
     "name": "A".repeat(10000),  // строка из 10 000 символов «A»
     "price": 1000,
     "statistics": {
       "contacts": 3,
       "likes": 500,
       "viewCount": 1523
     }
   }
2.Проверьте статус‑код и содержимое ответа.

Ожидаемый результат

Статус‑код: 400 Bad Request

Сообщение: «Длина поля name превышает допустимый лимит (например, 255 символов)»

Фактический результат

Статус‑код: 200 OK

Объявление создаётся, поле name сохраняется целиком (10 000 символов)

Возможны побочные эффекты:

переполнение хранилища;

замедление запросов на получение объявлений;

ошибки при отображении в интерфейсе.

Скриншоты/логи

-

Приоритет
High

Статус
Open

Дополнительные замечания

Необходимо определить и документировать максимальный лимит длины для поля name.

Проверить, есть ли валидация на других строковых полях (например, sellerID как строка, комментарии и т.п.).

Убедиться, что база данных и слои приложения корректно обрабатывают длинные строки без SQL‑инъекций или иных уязвимостей.


### Что было исправлено/дополнено:
1. **Полное тело запроса** — добавлен пример JSON с длинным `name`.
2. **Ожидаемый результат** — уточнено сообщение об ошибке и упомянут типичный лимит (255 символов).
3. **Фактический результат** — описаны возможные побочные эффекты.
4. **Дополнительные замечания** — предложены шаги для анализа и предотвращения рисков.
